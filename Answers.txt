Git Nedir?
Bir dosya veya proje üzerinde birden fazla kişi değişiklik yapacaksa bu değişikliklerin takibini 
yaptığımız versiyon kontrol sistemine verilen addır.
-----------------------------------------------------------------------------------------------------------------------------
"git pull" ile "git fetch" komutlarının farkı nedir?
Git fetch ile uzak sunucudaki değişiklikleri kendi local repomuzun haberdar olmasını sağlarken 
güncellemeleri değiştirmez.
Git pull ise uzak sunucudaki değişiklikleri kendi local repomuza alarak değişiklikleri günceller.
-----------------------------------------------------------------------------------------------------------------------------
Eğer takım arkadaşımız "kodlarımı gönderdim, benim geliştirmemin üzerine 
devam et" derse ve gönderdiği kodları "git pull" ile lokalimize alamıyorsak 
nerelerde hata yapılmış olabilir? 
-Arkadaşımız yazdığı kodları remote sunucuda bulunan repoya göndermemiştir. Yani git push 
komutunu kullanmamıştır.
-Arkadaşımız kodlarını commitlememiş olabilir.(commitledemen de pushlayamayacağı için push 
atamamıştır)
-Biz kodda değişiklik yapıp commitlemediğimiz için pull edemiyor olabilir.
-----------------------------------------------------------------------------------------------------------------------------
“git fetch origin" komutundaki "origin" neye karşılık gelmektedir?
Origin uzak sunucuya tanımlanan bir isimdir.
-----------------------------------------------------------------------------------------------------------------------------
"HEAD" kelimesi neyi temsil etmektedir?
HEAD kelimesi en son commiti temsil eder.
-----------------------------------------------------------------------------------------------------------------------------
"Staging Area" ya da "Index" diye isimlendirilen bölge tam olarak ne demektir?
Staging Area, ara bölge anlamına gelir. git add komutunu çalıştırdıktan sonraki dosya değişiklikleri buraya gönderilir.
-----------------------------------------------------------------------------------------------------------------------------
"Untracked file" ne demektir?
Untracked file, git tarafından takip edilmeyen dosya anlamına gelir. Yeni bir dosya oluşturduğumuzda ortaya çıkar.
-----------------------------------------------------------------------------------------------------------------------------
".git" klasörünü silersek ne olur?
.git klasörünü silersek o dosyadaki git özelliklerini kaybetmiş oluruz.
-----------------------------------------------------------------------------------------------------------------------------
Kendi lokalimizde her "git init" komutunu kullanıdığımızda otomatik olarak "ReadMe.md" dosyası oluşturulmasını istiyorsak ne yapmalıyız?
Bir template oluşturup bu templatein içine Readme.md dosyası ekleriz. Ve bu template'i git init --template=... şeklinde farklı git klasörlerimize ekleyerek otomatik olarak Readme.md dosyasını oluşturmuş oluruz.
-----------------------------------------------------------------------------------------------------------------------------
Git konusunda bahsi geçen "branch" yapısı nedir? Bize ne sağlar?
Branch yapısı, farklı geliştirme ortamlarını yönetmek için kullanılır. master ya da main adıyla kullanılan branch e ana dal ismini verebiliriz. Kendimiz farklı yapılarda çalışma yapacaksak kendi branchimizi oluşturup bu dal üzerinde çalışmalarımızı yürütebiliriz.
-----------------------------------------------------------------------------------------------------------------------------
Sıfırdan bir "branch" nasıl oluşturabiliriz?
git branch <branch_name> ---- git checkout -b <branch_name> komutlarıyla sıfırdan bir branch oluşturulabilir.
-----------------------------------------------------------------------------------------------------------------------------
Var olan bir "branch"e nasıl geçebiliriz?
git checkout <branch_name> komutuyla farklı bir branche geçiş yapılabilir.
-----------------------------------------------------------------------------------------------------------------------------
"git clone" komutunu kullanırken belirli bir spesifik branch'i sadece çekmek istiyorsak nasıl yapabiliriz?
git clone --branch <branch_name>
-----------------------------------------------------------------------------------------------------------------------------
"Merge conflict" ne demektir?
Farklı branchler üzerinde işlemler gerçekleştirildikten sonra branchlerin "Merge" işlemiyle birleştirilmesi gerekir. Ancak bu sırada;
-aynı dosya üzerinde aynı satırlarda değişiklik oluştuysa
-birisi dosyayı silmişken başka kullanıcı bu dosya üzerinde değişiklik yaptıysa merge conflict meydana gelir.
-----------------------------------------------------------------------------------------------------------------------------
"git log" komutu ile hangi bilgileri görebiliriz?
Commit Hash: Her bir commit için benzersiz bir id
Yazar Bilgisi: Değişikliğin kim tarafından yapıldığı bilgisi
Tarih ve Saat: Commitin ne zaman yapıldığı
Commit Mesajı: Yapılan değişikliği açıklayan bir commit mesajı
-----------------------------------------------------------------------------------------------------------------------------
"git diff" ile kaç farklı iki durumun arasındaki değişiklikleri görebiliriz?
İki durum arasındaki farkları görmek için komutu kullanılır.
